
- 三个参数都可省略：省"起"从头、省"止"到尾、省"步"默认1

## 第6节

字典的增删改查：

| 操作   | 方式                      | 示例                      | 说明            |
| ---- | ----------------------- | ----------------------- | ------------- |
| 增/改  | `d[k]=v`                | `d["赵六"]=80`            | 键不存在则新增，存在则修改 |
| 查    | `d[k]` / `d.get(k)`     | `d["张三"]`               | get 更安全       |
| 删    | `del d[k]` / `d.pop(k)` | `del d["李四"]`           | pop 返回被删的值    |
| 批增   | `d.update(d2)`          | `d.update({"x":1})`     | 合并另一个字典       |
| 遍历键  | `d.keys()`              | `for k in d:`           | 最常用简写         |
| 遍历值  | `d.values()`            | `for v in d.values():`  |               |
| 遍历键值 | `d.items()`             | `for k,v in d.items():` | **最常用**       |

四大需求：

|需求|选谁|理由|
|---|---|---|
|按顺序存、要能改|`list`|有序可变|

|用名字查值|`dict`|键值映射|
|去重 / 判断在不在|`set`|唯一 + O(1)查找|

## 第8节

查找速度：

|容器|查找方式|速度|复杂度|
|---|---|---|---|
|`list`|从头挨个比对|**慢**|O(n)：元素越多越慢|
|`dict`|按键哈希定位|**快**|O(1)：几乎不随元素增多变慢|
|`set`|哈希定位|**快**|O(1)|

- **list**：找 `x in lst` 要从第1个挨个比，最坏要比 n 次。1万个元素查1次，要比1万次。
- **dict/set**：基于**哈希表**，通过哈希函数直接算出 x 的位置，一步命中，几乎和元素数量无关。
