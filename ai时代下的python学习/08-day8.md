 ## 第9节 · match-case（Python 3.10+ 结构化模式匹配）

### 基本知识重点

- Python 3.10 引入 `match-case`，类似其他语言的 switch，但更强大（支持结构匹配）
- `case _` 是通配符，兜底匹配

```python
# 基本等值匹配
def day_cn(n):
    match n:
        case 1: return "周一"
        case 2: return "周二"
        case 3: return "周三"
        case _: return "未知"     # _ 是通配，兜底

# 多值合并匹配
def weekend(n):
    match n:
        case 6 | 7: return "周末"   # | 表示或
        case _: return "工作日"
```

### 选择标准

| 场景 | 推荐 |
|:---|:---|
| 纯等值查值 | 字典更简洁 |
| 需要绑定变量/解构 | match-case |
| 需要守卫条件 `if` | match-case |

> **建议**：纯查值用字典；需要"匹配结构 + 提取变量"才用 match-case。本册只要求认识，不深入。

## 第11节 · assert：断言自检

### 基本知识重点

- **语法**：
  - `assert 条件` — 条件为假则抛 `AssertionError`
  - `assert 条件, "错误信息"` — 带提示

```python
def divide(a, b):
    assert b != 0, "除数不能为0"   # 开发期自检
    return a / b
```

- **用途**：
  - **自检**：写完函数立刻验证结果对不对
  - **契约**：函数入口检查参数合法性
  - **教学**：本册所有练习用 assert 自检

### 易错点

- `assert` 在 `python -O` 优化模式下会被**忽略**
- **不要用它做生产环境校验**，生产环境用 `if + raise`

> **区分**：assert 失败说明"我假设错了"，是开发期工具，不是用户期错误处理。
