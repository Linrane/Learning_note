## 第1节

程序处理的是数据。分不清类型，就会在转换和运算上摔跟头——这类 bug 往往安静潜伏，直到特定输入触发。

变量的命名规则：

| 规则       | 好               | 差                        |
| -------- | --------------- | ------------------------ |
| 有意义      | `student_count` | `x1`                     |
| 小写 + 下划线 | `total_price`   | `TotalPrice`（PEP8 不推荐驼峰） |
| 不以数字开头   | `score_1`       | `1score`                 |
| 不用保留字    | `user_name`     | `class`, `for`, `print`  |
## 第2节

四种基本类型

|类型|英文|示例|适用场景|
|---|---|---|---|
|整数|`int`|`42`, `-7`, `0`|计数、索引、无小数运算|
|浮点数|`float`|`3.14`, `0.1`|科学计算、价格、测量值|
|字符串|`str`|`"hello"`, `'你好'`|文本、标识符、需拼接展示|
|布尔值|`bool`|`True`, `False`|条件判断、开关状态|

使用type()可以查看变量类型

典型误用：

| 用                 | 后果                           |
| ----------------- | ---------------------------- |
| 字符串 `"3" + 5`     | `TypeError`：不能 str 与 int 直接加 |
| 用 `float` 做金额精确计算 | 二进制浮点误差（如 `0.1+0.2`）         |
| 把 `bool` 当 0/1 滥用 | 可读性差，易混淆                     |
| 空字符串 `bool("")`   | 为 `False`，非空为 `True`         |

（字符串和int没法直接相加，float没法精确相加，bool不是0和1的代名词，bool("")为空！）

## 第4节

常用函数的转换：

| 函数         | 作用   | 典型失败                    |
| ---------- | ---- | ----------------------- |
| `int(x)`   | 转整数  | `int("3.14")` 报错        |
| `float(x)` | 转浮点  | `float("abc")` 报错       |
| `str(x)`   | 转字符串 | 极少失败                    |
| `bool(x)`  | 转布尔  | `0`、`""`、`None` 为 False |

转换方案的对照：

|方案|代码|结果|
|---|---|---|
|直接 float|`float("3.14")`|`3.14` ✓|
|先 int 再 float|`float(int("3.14"))`|报错 ✗|

|方案|代码|结果|
|---|---|---|
|str 转换|`"答案是" + str(42)`|正确 ✓|
|直接拼接|`"答案是" + 42`|TypeError ✗|
|f-string|`f"答案是{42}"`|正确 ✓（推荐）|

第三种还不会把结果写死，是很推荐的一种方式。
