 ## 第1节 · if 的本质：条件为真就执行

### 基本知识重点

- `if` 后跟**布尔表达式**，为 `True` 才执行缩进块
- **缩进决定归属**，4 空格是规范（不要混用 Tab）
- `if` 末尾必须有冒号 `:`
- Python 中以下值视为**假（False）**：`False`、`0`、`0.0`、空字符串 `""`、空容器 `[]`/`{}`/`()`/`set()`、`None`
- **其余全是真**。`if lst:` 表示"lst 非空就执行"，比 `if len(lst)>0:` 更 Pythonic

### 易错点

| 错误           | 后果                 | 正确         |
| :----------- | :----------------- | :--------- |
| 忘冒号 `if x>0` | `SyntaxError`      | `if x>0:`  |
| 混用 Tab 和空格   | `IndentationError` | 全用 4 空格    |
| `=` 当 `==`   | `if x=5:` 报错       | `if x==5:` |

> **铁律**：判断相等用 `==`，赋值用 `=`。

## 第2节 · if/else：二选一

### 基本知识重点

- `else` 是"以上都不满足"的兜底，**不需要写条件**
- `else` **必须有对应的 if** 才能用

### 易错点

| 错误                      | 后果            |
| :---------------------- | :------------ |
| `else` 后加条件 `else x>0:` | `SyntaxError` |
| `else` 单独出现没有 `if`      | `SyntaxError` |
| 缩进错位让本该属于 else 的语句跑到外面  | 逻辑错乱          |
|                         |               |

## 第3节 · if/elif/else：多分支（互斥匹配）

### 基本知识重点

- `elif` 是 `else if` 的缩写
- 分支**从上到下顺序匹配，命中一个就停止**，后续不再判断
- **条件顺序很重要——大阈值放前面**

```python
# ❌ 错误顺序
if score >= 60: grade = "C"
elif score >= 80: grade = "B"
elif score >= 90: grade = "A"

# ✅ 正确顺序
if score >= 90: grade = "A"
elif score >= 80: grade = "B"
elif score >= 60: grade = "C"
else: grade = "D"
```

- **互斥**：分支之间互不相交，一次只走一条（如成绩分档）→ 用 `elif`
- **顺序**：多个独立 if 各自判断，可能多条都走 → 用多个 `if`

> **口诀**：互斥用 elif，独立用多个 if。

## 第4节 · 比较运算符与链式比较

### 基本知识重点

| 运算符  | 含义   | 示例       | 结果    |
| :--: | :--- | :------- | :---- |
| `==` | 等于   | `5 == 5` | True  |
| `!=` | 不等   | `5 != 3` | True  |
| `>`  | 大于   | `5 > 3`  | True  |
| `<`  | 小于   | `5 < 3`  | False |
| `>=` | 大于等于 | `5 >= 5` | True  |
| `<=` | 小于等于 | `5 <= 5` | True  |

- **链式比较**（Python 特色）：`0 < x < 10` 等价 `0 < x and x < 10`

### 易错点

- `==` 比较的是**值**，不是身份（身份用 `is`）
- 浮点数比较要小心：`0.1 + 0.2 == 0.3` 是 `False`，用 `abs(a-b) < 1e-9`
- 字符串比较按字典序：`"A" < "a"` 为 True

## 第5节 · 嵌套 if：组合条件与扁平化

### 基本知识重点

- **何时嵌套**：条件有**层级依赖**（先判断 A，A 成立才需要判断 B）
- **避免过深——超过 3 层就重构**，用提前 return 扁平化

```python
# ❌ 嵌套过深
if a:
    if b:
        if c:
            if d:
                do()

# ✅ 提前 return（扁平化）
if not a: return
if not b: return
if not c: return
if not d: return
do()
```

- **并列无依赖** → `and` 组合更扁平
- **层级依赖**（B 只在 A 成立时才有意义）→ 嵌套更清晰

> **口诀**：并列用 and，依赖用嵌套，过深用提前 return。

## 第6节 · 组合条件：and / or / not 与短路

### 基本知识重点

| 运算符 | 含义 | 真值表 | 示例 |
|:---:|:---|:---|:---|
| `and` | 且 | 都真才真 | `age>=18 and has_ticket` |
| `or` | 或 | 一真即真 | `is_vip or score>90` |
| `not` | 非 | 取反 | `not is_banned` |

- **短路求值**：
  - `and`：前为假则**不判断后面**（结果已定）
  - `or`：前为真则**不判断后面**（结果已定）

```python
# 短路避免除零
if b != 0 and a/b > 1: ...     # b==0 时不会执行 a/b
# 短路做默认值
name = user_input or "匿名"     # user_input 为空则用"匿名"
```

- **优先级**：`not > and > or`

### 易错点

- 复杂条件不加括号容易错：`if a or b and c:` 等价 `if a or (b and c)`
- 建议复杂条件一律加括号

> **铁律**：复杂条件一律加括号，别让优先级替你做决定。

## 第7节 · 三元表达式：一行搞定简单赋值

### 基本知识重点

- **语法**：`值1 if 条件 else 值2`——条件真取值1，假取值2

```python
# 普通写法
if score >= 60:
    msg = "及格"
else:
    msg = "不及格"

# 三元表达式
msg = "及格" if score >= 60 else "不及格"
```

### 适用与不适用

| 场景 | 用三元？ |
|:---|:---:|
| 简单二选一赋值 | 推荐 |
| 有副作用（print、多语句） | 用 if/else |
| 三层以上嵌套 | 难读，用 if/elif |
| 仅判断不赋值 | 用 if |

- 嵌套三元**慎用**：`sign = "正" if n > 0 else ("零" if n == 0 else "负")`
- 四选一就别用三元，改 if/elif

> **铁律**：三元只用于"一眼能看懂的赋值"。一旦要嵌套两层以上，立刻改 if/elif。

## 第8节 · 字典映射替代多分支

### 基本知识重点

- 当分支是"按一个值查一个结果"且分支很多时，用字典替代 `if/elif`

```python
# if/elif 写法
def day_cn(n):
    if n==1: return "周一"
    elif n==2: return "周二"
    elif n==3: return "周三"
    elif n==4: return "周四"
    elif n==5: return "周五"
    else: return "未知"

# 字典映射
DAYS = {1:"周一",2:"周二",3:"周三",4:"周四",5:"周五"}
def day_cn(n):
    return DAYS.get(n, "未知")
```

### 选择标准

| 分支类型 | 推荐方式 |
|:---|:---|
| **等值匹配**（n==某值） | 字典映射 |
| **范围判断**（score>=90） | if/elif |
| **复杂条件**（多变量组合） | if/elif |

### 字典映射的优势

- **简洁**：新增分支只加一行字典项
- **可配置**：字典可从配置文件加载，分支逻辑变数据
- **可扩展**：值可以是函数（lambda），实现策略模式
