# 容器核心：列表List和字典Dict

## 第1节

**变量只能存一个值，容器能存一组值。**

一个变量装不下（一对一，逻辑为映射），调用起来太麻烦，多了还容易崩溃，我需要一个就能管一堆的！

四大容器：

|容器|特征|一句话记忆|典型场景|
|---|---|---|---|
|`list` 列表|有序、可变、可重复|**能改能重复**|一组成绩、一组名字|
|`tuple` 元组|有序、不可变、可重复|**锁定不能改**|固定坐标、函数多返回值|
|`dict` 字典|键值映射、键唯一|**靠键找值**|姓名→成绩、配置项|
|`set` 集合|无序、元素唯一|**只管有没有**|去重、交集并集|

列表能改能重复，元组锁定不能改，字典靠键要找值，集合只管有没有。

set的查找会引发typeerror报错，没法索引，如果既想要去重特性，又想要他按特定位置来，但是
我们只要先转成list就好了。

```python
s = {1,2,3}
lst= list(s)
print(lst[0])
```

例子，但这个转换后的顺序，不一定会和一开始定义的顺序一致。

但为什么最后结果还是1？因为set底层是哈希表，对于小范围的参数，哈希表和数字本身一致，储存刚好由小到大，顺序1，2，3，所以第一个元素是0

什么是哈希表？

键值映射的数据结构，底层是靠哈希函数将任意的数字转换成数字下标，直接定位存储位置，实现“近乎瞬间查找”。

你手机通讯录里"联系人列表"，更适合用 list 还是 dict？为什么？

dict！逻辑是键值对，用哈希表，瞬间查找。

三个易错点：

|错误|后果|正确做法|
|---|---|---|
|索引从 1 开始数|取错值|**从 0 开始**，最大索引 = `len-1`|
|`scores[5]`（长度5）|`IndexError` 越界|最大只能 `scores[4]`|
|忘记 `len()` 用 `count`|报错|取长度用 `len()`|

**真实场景**：取"最后一个元素"用 `lst[-1]`，比 `lst[len(lst)-1]` 简洁得多，这是 Python 程序员的习惯。

## 第3节

方法速查表

| 操作   | 方法             | 示例                  | 返回   | 说明              |
| ---- | -------------- | ------------------- | ---- | --------------- |
| 增(尾) | `append(x)`    | `lst.append(70)`    | None | 末尾追加，**最常用**    |
| 增(位) | `insert(i,x)`  | `lst.insert(0,99)`  | None | 在索引 i 处插入       |
| 增(批) | `extend(lst2)` | `lst.extend([1,2])` | None | 把另一个列表拼进来       |
| 查(位) | `lst[i]`       | `lst[0]`            | 元素   | 按索引取值           |
| 查(值) | `x in lst`     | `88 in lst`         | bool | 判断元素在不在         |
| 查(位) | `lst.index(x)` | `lst.index(75)`     | 索引   | 找不到报 ValueError |
| 查(次) | `lst.count(x)` | `lst.count(88)`     | 次数   | 统计出现次数          |
| 改    | `lst[i]=x`     | `lst[0]=50`         | -    | 按索引赋值           |
| 删(值) | `remove(x)`    | `lst.remove(60)`    | None | 删**首个**等于 x 的   |
| 删(位) | `pop(i)`       | `lst.pop(0)`        | 被删元素 | 弹出并返回，不传 i 弹末尾  |
| 删(全) | `del lst[i]`   | `del lst[0]`        | -    | 按索引删            |
| 清空   | `clear()`      | `lst.clear()`       | None | 清空列表            |
