## 1.练习

代码不是写出来好看的，是跑出来能验收的。

验收=能用？不能！

python的基础用法：

```python
print("I am Linrane,I love python.")
print("python","summer"."industry","technology")
```

注：不可混用：print(2python)

还可以用于结果导向：

```python
print("loading……")
print("loading"+"……")#相加起来了
print("finish!!!")
```

常见报错：

| 报错                | 原因           | 修复                          |
| :---------------- | :----------- | :-------------------------- |
| `SyntaxError`     | 引号不配对、用了中文括号 | 检查 `'` `"` `()`             |
| `NameError`       | 变量未定义就使用     | 先赋值再 print                  |
| 输出不是预期文字          | 逗号与加号混用理解错误  | `+` 只用于字符串拼接，数字需 `str()` 转换 |
| ZeroDivisionError | 分母为0         | 分母1上的0检查后改成其他的              |

## 2.注释的标准写法

注释是写给自己和别人看的，不是给解释器看的。

```python
#这是一行注释
"""
这是
多行
注释
"""
```

同时，如果你暂时不想运行某行，也可以这么做，但是还要加一些说明（注：被注释的代码不会被运行）

| 该写           | 不该写                  |
| :----------- | :------------------- |
| 解释非显而易见的业务规则 | `# 把 x 加 1`（代码已经说明了） |
| 标注已知限制或临时方案  | 大段注释掩盖烂代码            |
| 说明复杂公式的来源    | 把注释当聊天记录             |

```python
#A:注释是否有价值
x = 10
x = x + 1 #这里是x加1，记住

#B:有价值？
is_probation = True #员工处于试用期
#业务规则：试用期员工不发放年终奖，即使入职年份为12月
if is_probation : # 值为True
	bonus = 0
print("B 执行后bonus = ",bonus)
```

## 3.如何下精确指令

工具不是读心者，给的指令越模糊，产出越随机

五要素：

| 要素       | 问题       | 示例                                                        |
| :------- | :------- | :-------------------------------------------------------- |
| **任务目标** | 要做什么？    | 写一个函数，统计列表中正数的个数                                          |
| **输入**   | 参数是什么类型？ | 参数 `nums: list`，元素为 int 或 float                           |
| **输出**   | 返回什么？    | 返回 int，表示正数个数                                             |
| **约束**   | 有什么限制？   | 不使用第三方库；空列表返回 0                                           |
| **验收标准** | 怎样算通过？   | `count_positive([1,-2,3])` 返回 2；`count_positive([])` 返回 0 |

模糊vs精确

|模糊（差）|精确（好）|
|:--|:--|
|帮我写个处理数据的程序|用 Python 读取同目录下 `sales.csv`，打印前 5 行，使用 pandas|
|做个计算器|写函数 `add(a, b)` 返回两数之和，参数为 int，不处理非数字输入|
|优化一下这段代码|在不改变输出的前提下，把下面函数的行数从 20 行减到 10 行以内：[贴代码]|

课堂作业：帮我写一个贪吃蛇小游戏（模糊）——（精确）请使用Python的Pygame库编写一个600x400像素、15FPS、方向键控制、含计分与游戏结束重开功能的单文件贪吃蛇游戏。
## 4.好代码和坏代码

|维度|好代码|坏代码|
|:--|:--|:--|
|**可读性**|变量名有意义|`a`, `tmp`, `data1`|
|**可预测**|同样输入同样输出|依赖全局变量，结果随运行顺序变|
|**边界处理**|考虑空输入、异常|默认输入永远正常|
|**适度**|能读懂的最简实现|一行写 200 字符|
