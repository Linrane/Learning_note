易错点：

| 错误                       | 后果          | 正确                               |
| ------------------------ | ----------- | -------------------------------- |
| 遍历字典直接 `for v in d`      | 拿到的是键不是值    | 用 `d.values()` 或 `d.items()`     |

    up += ch.upper()

## 第2节

3种形式：

```python
range(5)          # 0,1,2,3,4         只给止
range(2, 6)       # 2,3,4,5           起:止(不含止)
range(0, 10, 2)   # 0,2,4,6,8         起:止:步
range(10, 0, -1)  # 10,9,...,1        倒序
range(0, 10, 3)   # 0,3,6,9           步长3
```

range不是列表！！！：

```python
range(5)          # range(0,5)  它是"惰性序列"，不占内存
list(range(5))    # [0,1,2,3,4] 需要时才转成列表
```

注：range 惰性生成，要 100 万个数也不占内存（range 只记【生成数字的公式规则】，不存全部数字，用到才算，省内存），这是它比 `list(range(n))` 高效的原因。

（惰性就是懒加载，不提前把所有数字算出来，存进内存，只有用到它的时候才临时算出当前这样一个数字）

常见用法：

| 场景     | 写法                                    |
| ------ | ------------------------------------- |
| 重复 n 次 | `for _ in range(n):`                  |
| 取索引    | `for i in range(len(lst)):`           |
| 倒序遍历   | `for i in range(len(lst)-1, -1, -1):` |
| 跳步     | `for i in range(0, n, 2):`            |

易错点：

- `range(1, 10)` 是 1~9，**不包含 10**
- `range(10, 0)` 是空的（默认步长1，10→0 不可能），倒序必须 `range(10,0,-1)`
- `range(5)[4]` 是 4，但 `range(5)[5]` 越界

练习：

```python
#range实操
#1~100求和
s = sum(range(1,101))
assert s == 5050
print("pass!")
```

## 第3节

基本：

```python
n = 5
while n > 0:
    print(n)
    n -= 1          # 必须改变条件，否则死循环
# 5,4,3,2,1
```

三要素：

1. **初始化**：`n = 5`
2. **条件判断**：`while n > 0`
3. **条件更新**：`n -= 1`（漏掉就死循环！）

for vs while：

| 维度    | for       | while          |
| ----- | --------- | -------------- |
| 次数    | 已知 / 遍历容器 | 未知             |
| 典型场景  | 遍历列表、固定次数 | 读到文件尾、用户输入quit |
| 死循环风险 | 几乎不会      | 容易（忘更新）        |
| 更新    | 自动        | 手动             |
（知道次数用 for，不知道次数用 while）

练习：

```python
#while basci
n = 5
out = []
while n > 0:
    out.append(n)
    n -= 1
assert out == [5,4,3,2,1]
print("ALL PASS!!!")
```

```python
#while 次数未知
#不断除2直到小于1，统计次数
n = 100
count = 0
while n >= 1:
    n = n/2
    count += 1
assert count == 7
print("PASS!")
```

## 第4节

两个语句：

|语句|作用|一句话|
|---|---|---|
|`break`|**立即结束整个循环**|"我不干了"|
|`continue`|**跳过本轮剩余，进入下一轮**|"这轮跳过，下轮继续"|

<img width="519" height="642" alt="image" src="https://github.com/user-attachments/assets/42908eb7-26be-4531-8432-c8797ea36b68" />

易错点：

|错误|后果|
|---|---|
|`continue` 后还有更新语句|更新被跳过，可能死循环|
|`break` 写错位置|提前退出，漏处理|
|在 while 里 `continue` 前忘更新|死循环|

练习：

```python
#break找目标
for i in range(1,21):
    if i % 3 == 0:
        first = i
        break
assert first == 3
print("pass!")
```

```python
#continue跳过
data = [1,-2,3,-4,5]
total = 0
for n in data:
    if n < 0:
        continue
    total += n
assert total  == 9
print("pass")
```
